## 1. 本計画の目的

基本設計書（IPA 準拠）を入力として、IPA（独立行政法人情報処理推進機構）のソフトウェア開発ガイドラインに準拠した**詳細設計書**を作成する。詳細設計書は、基本設計書の「外部仕様」を「内部仕様（実装可能な設計）」に落とし込むことを目的とする。

基本設計書が「何をするか（What）」を定義したのに対し、詳細設計書は「どのように実現するか（How）」を定義する。

### 入力ドキュメント

| ドキュメント | パス | 役割 |
|---|---|---|
| 基本設計書 | `docs/basic_design/` (01〜12 + 付録) | 外部仕様・機能設計（直接入力） |
| アーキテクチャドキュメント | `docs/architecture/` (01〜08) | 設計判断の根拠、静的・動的ビュー |
| 計画書 | `docs/gridtwin_lab_plan.md` | プロダクト定義、ロードマップ |
| 基本設計書作成計画 | `docs/basic_design_plan.md` | ID 体系・トレーサビリティ方針 |

### 基本設計書との関係

```
計画書 → アーキテクチャドキュメント → 基本設計書 → 【詳細設計書】 → 実装
  (Why)        (What + Why)            (What)       (How)          (Code)
```

## 2. 詳細設計書の構成（IPA 準拠）

IPA のソフトウェア詳細設計書ガイドライン（プログラム設計書、モジュール設計書に相当）を参考に、gridflow の特性（Python 3.11+、CLI ベース、Docker 環境、Clean Architecture）に合わせて構成する。

### 基本設計書（What）→ 詳細設計書（How）の詳細化対応

基本設計書と詳細設計書は 1:1 ではなく N:M の関係を持つ。1つの基本設計章が複数の詳細設計章に展開され、逆に複数の基本設計章が1つの詳細設計章に統合される場合がある。

| 基本設計書の章 | 詳細設計書での展開先 | 詳細化の内容 |
|---|---|---|
| 第1章 要求一覧 | 第1章 要件一覧 | DD-xxx ID 追加、FN-xxx → モジュール → クラス対応表追加 |
| 第2章 システム方式設計 | 第2章 モジュール構成 + 第9章 設定管理 + 第11章 ビルド・デプロイ | Docker 構成 → Dockerfile・Compose 具体設計、ネットワーク設定 |
| 第3章 機能設計 | 第3章 クラス設計 + 第4章 処理フロー設計 + 第8章 エラー設計（FN-008 ロギング） | 機能 → クラス・メソッド（IPO形式）・処理フロー（FN-001〜FN-009 全件） |
| 第4章 CLI 設計 | 第3章 3.7 CLI クラス設計（出力仕様・状態遷移図含む） + 第4章 処理フロー設計（コマンドフロー） | コマンド体系 → CLI パーサー・ハンドラー・出力フォーマット実装設計 |
| 第5章 データ設計 | 第3章 3.4 CDL クラス + 第6章 データ詳細設計 + 第9章 設定管理（5.4 設定ファイル仕様） | CDL スキーマ → 型定義・制約・バリデーション・エクスポータ設計 |
| 第6章 外部IF設計 | 第3章 3.5 Connector（REST API仕様含む） + 第3章 3.8 Plugin API + 第4章 4.12-4.14 | Protocol 仕様 → 具体クラス・API仕様・シーケンス（IF-01〜IF-07 全件） |

## 7. 作成方針

1. **基本設計書との一貫性**: 用語・ID は基本設計書および ACDM に準拠する。詳細設計書独自の用語は付録で定義する
2. **トレーサビリティ第一**: 全設計項目が DD-xxx → REQ-xxx → BG-xxx まで追跡可能であること
3. **実装可能な粒度**: クラス・メソッドシグネチャ・型定義レベルまで具体化する。ただし実装コードそのものは書かない（疑似コード・型ヒント付きシグネチャまで）
4. **IPA 準拠 + gridflow 最適化**: IPA のプログラム設計書・モジュール設計書の標準構成を基本とし、Python / Clean Architecture / Docker 環境に合わせて読み替える
5. **章単位の独立作成**: 各章は独立してレビュー可能な粒度とし、Phase 順に完成させる
6. **Mermaid 図の活用**: 以下の図を Mermaid 記法で記述し、ドキュメント内で直接レンダリング可能とする
   - クラス図（classDiagram）: 第3章
   - シーケンス図（sequenceDiagram）: 第4章
   - アクティビティ図（flowchart）: 第4章（分岐・並行処理）
   - 状態遷移図（stateDiagram-v2）: 第5章
   - ER 図（erDiagram）: 第6章
7. **IPO 形式の徹底**: 全メソッドの定義に Input（引数・型）→ Process（処理概要）→ Output（戻り値・型・例外）を明記し、実装者が迷わない粒度とする
