## 11.1 Dockerfile 設計（REQ-C-001, REQ-C-002）
マルチステージビルド。コメント付きDockerfile（コードブロック）を2種:
- gridflow-core: Python 3.11-slim → builder(uv install) → runtime(非rootユーザー)
- opendss-connector: Python 3.11-slim → builder(uv install + OpenDSSDirect.py) → runtime
セキュリティ: 非rootユーザー(gridflow:1000)、不要パッケージ削除、.dockerignore

## 11.2 Docker Compose 構成詳細（REQ-C-002）
完全なdocker-compose.yml（コメント付き）:
- services: gridflow-core(ports:8888), opendss-connector(内部のみ)
- networks: gridflow-net(bridge)
- volumes: shared-data, scenario-packs(bind mount)
- healthcheck定義、depends_on、restart policy

## 11.3 CI/CD パイプライン詳細（M-06）
GitHub Actions workflow YAML。6ステージ:
1. lint: ruff check + ruff format --check
2. typecheck: mypy --strict
3. test: pytest --cov (ユニットテスト)
4. integration: docker compose -f docker-compose.test.yml up
5. build: docker buildx (AMD64+ARM64)
6. publish: PyPI + Docker Hub（タグ付与時のみ）
トリガー: push/PR(1-3), PR to main(4), main merge(5), tag(6)

## 11.4 パッケージング・配布設計
pyproject.toml構成（コードブロック）:
- [project]: name, version, requires-python=">=3.11", dependencies
- [build-system]: hatchling
- [tool.ruff], [tool.mypy], [tool.pytest]
配布フロー: tag作成→CI→PyPI publish + Docker Hub push

## 11.5 バージョニング戦略（M-07）
SemVer 2.0.0:
- PATCH(x.x.X): バグ修正、後方互換
- MINOR(x.X.0): 機能追加、後方互換
- MAJOR(X.0.0): 破壊的変更、移行ツール必須
リリースフロー: feature branch → PR → main merge → tag → release
