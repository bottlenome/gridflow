# 第1章 要求一覧

本章では、計画書およびアーキテクチャドキュメントから抽出した全要求を ID 付きで一覧化し、トレーサビリティの基盤を構築する。

## 更新履歴

| 版数 | 日付 | 変更内容 |
|---|---|---|
| 0.1 | 2026-04-01 | 初版作成 |

---

## 1.1 要求 ID 体系

| 接頭辞 | 分類 | 出典 |
|---|---|---|
| `REQ-B-xxx` | ビジネス要求 | アーキテクチャドキュメント BG-1〜BG-4 |
| `REQ-F-xxx` | 機能要求 | 計画書 P0/P1/P2 + アーキテクチャ FR-01〜FR-07 |
| `REQ-Q-xxx` | 品質要求 | アーキテクチャ QA-1〜QA-11 |
| `REQ-C-xxx` | 制約 | アーキテクチャ CON-1〜CON-6 |

---

## 1.2 ビジネス要求一覧

| ID | 要求名 | 内容 | 出典 |
|---|---|---|---|
| REQ-B-001 | E2E ループ高速化 | 電力系研究室の実験運用コスト（環境構築、連携コード作成、入出力整形、実験再実行、グラフ生成、表整理）を圧縮する | BG-1 |
| REQ-B-002 | 共同研究基盤として普及 | 導入研究室数をマイルストーンとして 10 → 50 → 100 研究室への普及を目指す | BG-2 |
| REQ-B-003 | 再現性の制度的担保 | 卒業と同時に実験が失われる問題を防ぎ、実験条件の再現性を Scenario Pack として制度化する | BG-3 |
| REQ-B-004 | 研究フロー・評価の標準化 | 研究フローと評価方法を標準化し、共通ベンチマークによる定量比較を可能にする | BG-4 |

## 1.4 品質要求一覧

| ID | 要求名 | 応答測定値 | 出典 |
|---|---|---|---|
| REQ-Q-001 | 導入容易性 | セットアップ完了時間 < 30分、成功率 > 90% | QA-1 |
| REQ-Q-002 | 初回利用効率 | Time to First Simulation < 1時間、CLI コマンド数 < 5 | QA-2 |
| REQ-Q-003 | 再現性 | 異なるマシン2台以上で同一 Scenario Pack・seed から同一結果 | QA-3 |
| REQ-Q-004 | 拡張性 | L2 Plugin 統合に必要なコード行数 < 100行 | QA-4 |
| REQ-Q-005 | ワークフロー効率 | ベースライン再現 < 1時間、アイデアから比較実験結果まで < 1日 | QA-5 |
| REQ-Q-006 | データエクスポート容易性 | データエクスポート → 論文図表の変換ステップ数 < 3 | QA-6 |
| REQ-Q-007 | ポータビリティ | AMD64 + ARM64 で全テスト通過 | QA-7 |
| REQ-Q-008 | 可観測性 | 計画書 10.1 の全 KPI がシステム内在の計測機構で取得可能 | QA-8 |
| REQ-Q-009 | LLM 親和性 | 構造化 I/O（JSON/YAML）、自己説明的エラー、Ubiquitous Language の一貫性 | QA-9 |
| REQ-Q-010 | 性能効率 | gridflow オーバーヘッドが外部ツール実行時間の 5% 以下 | QA-10 |
| REQ-Q-011 | 論文生産性 | gridflow 使用論文が年間 10本以上、L1 でもシナリオ比較論文が書ける | QA-11 |
