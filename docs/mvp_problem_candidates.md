## 候補 1: Volt-VAR 制御 (高 PV 浸透 + 雲影スパイク)

### 課題の核心
住宅街の配電フィーダーに屋根置き太陽光発電 (PV) が大量導入されると、晴天昼に逆潮流で電圧が許容上限 (例 1.05 pu) を超える違反が頻発。雲影で数秒スケールに発電量が 80% 振れるため、電圧も振動する。

各 PV インバータは無効電力 (Q) を吸収/注入できるが、Q をどう出すかの分散制御方式は以下のトリレンマ:

| 方式 | 限界 |
|---|---|
| 局所 droop (各機器単独) | 隣接機との干渉で hunting (振動) |
| 中央 MPC | 雲影 (秒) に対し計算 (数十秒) が遅い + 通信負荷大 |
| 分散 consensus | 通信遅延 50-500 ms で発散リスク |

### 学術ギャップ
**通信遅延を許しながら、局所単独より明確に良い分散制御** の理論が未確立。

### 解決のうれしさ
- 配電事業者: PV 接続容量 20-40% 増 (新規変電所投資回避)
- 需要家: PV 設置申請通過率向上
- 学問: 遅延ロバスト分散制御の応用先として希少

### 実装難度 (gridflow)
**低〜中**: pandapower で時系列 PF + 各バス Q 更新ループで再現可能。雲影は確率的時系列で生成。

## 候補 2: 仮想発電所 (VPP) の補助サービス提供 — 機器流出入 (churn) ロバスト性

### 課題の核心
VPP は数百〜数千の小規模 DER (EV、住宅蓄電池、エコキュート) を束ねて、系統運用者に補助サービス (周波数調整 5MW/30秒等) を契約供給する。問題は **メンバーが常時入れ替わる**:

- EV: 朝出発で離脱、夜帰宅で復帰
- 蓄電池: SOC 枯渇で離脱
- 故障/通信切断で離脱

契約は集約 SLA (= 全体で 5MW 出せ) なので、独立 1 台の離脱は吸収できるが、**バースト同時離脱** (夕方通勤・厳冬朝 等) で SLA 違反 → ペナルティ。

### 学術ギャップ
重尾分布 (heavy-tailed) の churn に対してロバストなリアルタイム機器入替アルゴリズム。確率モデルは平均ベースで重尾を外し、強化学習はブラックボックスで保証なし。

### 解決のうれしさ
- VPP 事業者: 過剰契約コスト 30-50% 削減
- 系統運用者: VPP を信頼可能担い手として補助サービス市場へ取込
- 学問: 「適応的アルゴリズム + 保証付き」路線の実問題

### 実装難度 (gridflow)
**中〜高**: pandapower 上に DER pool + churn シミュレータを追加実装する必要。周波数応答動力学は簡略化前提。

## 候補 3: 停電復旧時の遮断器投入順序

### 課題の核心
台風等で広域停電したフィーダー (放射状配電網、数百〜数千ノード) の修理完了後、遮断器を**どの順序で閉じるか**が問題。一斉投入すると変圧器・モーター負荷の同時起動で突入電流 (通常の 5-10 倍) が発生し、**再トリップ → 数十分ロス**。

### 学術ギャップ
状態 (各家庭の現在負荷) が**正確に分からない**ままの逐次決定。1 ステップ進むごとに観測が増えるが初期は情報極小。POMDP (部分観測マルコフ決定過程) クラスの難しい問題で実問題応用は希少。

### 解決のうれしさ
- 配電事業者: 復旧時間 20-30% 短縮 (顧客機会損失削減 + 規制ペナルティ回避)
- 顧客: SAIDI / SAIFI 改善
- 学問: POMDP の現実応用例

### 実装難度 (gridflow)
**高**: pandapower は静的 PF 主体で、突入電流のような秒以下過渡は不得手。本格には電磁過渡解析 (PSCAD 等) が必要 → gridflow スコープ外可能性大。
