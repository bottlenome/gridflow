# サブエージェントへの指示ルール

## 1. 作業指示書（WI）方式を使う

サブエージェントに直接「基本設計書を読んで」と指示しない。アーキテクト（メインエージェント）が必要情報を抽出し、作業指示書ファイルに埋め込む。

```
❌ 悪い例: 「/home/user/gridflow/docs/basic_design/03_function_design.md を読んで第3章を書いてください」
✅ 良い例: 「WI-00_common.md と WI-01_ch3_latter.md を読んで作業してください。他のファイルは読まないでください」
```

## 2. 作業指示書の構成

作業指示書は `docs/detailed_design/work_instructions/` に配置する。

- **WI-00_common.md**: 全設計者共通のルール（IPO形式、Mermaid図、更新履歴等）
- **WI-{番号}_{内容}.md**: タスク固有の指示。必要な情報（クラス定義、Protocol、計算式等）を直接埋め込む
- **WI-R{番号}_{内容}.md**: レビュー指示書。チェック観点・報告フォーマットを定義

## 3. 指示書に含めるべき内容

- **対象ファイルパス**: 作成/追記するファイルの絶対パス
- **必要な情報の直接埋め込み**: Protocol定義、属性一覧、メソッドシグネチャ、計算式等をコピペで埋め込む
- **成果物のフォーマット**: 冒頭テンプレート（更新履歴等）、セクション構成、記述形式（IPO形式等）
- **「他のファイルは読まない」の明示**: 必ず記載する

## 4. サブエージェントの起動ルール

- **最大並列数: 3エージェント**（レートリミット対策）
- **1エージェント1タスク**: 適切な分割粒度で分割する（下記「分割粒度の目安」参照）
- **読むファイルは最大2つ**: WI-00 + WI-{番号} のみ。追記タスクの場合は対象ファイルのReadも加わり最大3つ
- **プロンプトの定型文**:
  ```
  gridflow詳細設計者として作業してください。
  以下の2ファイルだけ読んで、指示に従ってファイルを作成してください。
  **他のファイルは一切読まないでください。**
  1. `/home/user/gridflow/docs/detailed_design/work_instructions/WI-00_common.md`
  2. `/home/user/gridflow/docs/detailed_design/work_instructions/WI-{番号}.md`
  ```

## 5. 分割粒度の目安

タスクが大きすぎるとタイムアウトやレートリミットで失敗する。以下を目安に分割する。

| 指標 | 目安 | 根拠 |
|---|---|---|
| セクション数 | 3〜5セクション/タスク | 5超でタイムアウトリスク増 |
| プロンプトに埋め込む情報量 | 2000文字以内 | WI-00(15行)+WI本体で合計2000文字が目安 |
| 生成される成果物の想定行数 | 100〜300行/タスク | 300行超は分割を検討 |
| 入力に必要なクラス/Protocol定義数 | 5定義以内 | 定義が多いと生成品質が落ちる |

**分割の判断フロー:**
1. セクション数が5以下 → 分割不要
2. セクション数が6以上 → 前半/後半に分割
3. 1セクション内のクラス数が8以上 → そのセクションだけで1タスク
4. 失敗（タイムアウト/レートリミット）した → さらに細かく分割して再試行

**実績値（本プロジェクト）:**
- 成功: WI-04(第5章, 4セクション) → 13Kトークン, 43秒, 176行
- 成功: WI-05(第7章, 6セクション) → 26Kトークン, 213秒, 655行
- 失敗: 第3章全体(9セクション) → タイムアウト
- 成功: 第3章3.1-3.4(4セクション) → 23Kトークン, 296秒, 519行

## 6. レビュアーの起動ルール

- **レビュー指示書（WI-R{番号}）を必ず先に読ませる**
- **チェック観点をチェックリスト形式で定義**: A(構造品質), B(設計原則), C(整合性), D(完成度)
- **報告フォーマットを指定**: ✅/⚠️/❌ 形式。問題なし項目はまとめて列挙、問題ありのみ個別報告
- **相互整合性レビュー（WI-R2）**: 章間ペアを明示し、チェック対象を限定する

## 7. 追記タスクの注意

既存ファイルに追記する場合:
- 「Readで既存内容を読み、末尾に追記した全体をWriteする」と明示
- 更新履歴に追記版を追加するよう指示

## 8. コミット・プッシュ

- サブエージェントの完了通知が来たらすぐにファイルサイズ確認 → コミット → プッシュ
- コミットメッセージに章番号・セクション範囲・主要内容を含める
- レビュー指摘のERROR修正は即座に対応し、別コミットで記録
