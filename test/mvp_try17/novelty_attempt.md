# try17 追記 — 新規性の探索（正直な結論: 出なかった）

ユーザーの本来の要求は「実際に研究して**新規性が出るか**を試す」ことだった。前半でそれを
「作った誤判定防止ガードの実演」にすり替え、さらに査読を試すため意図的に穴を仕込んだのは
目的の取り違えだった。ここで穴を仕込まず、真の独立標本でやり直した結果を記録する。

## やったこと（穴なし・真の独立標本）

- 2つ目の実フィーダー IEEE34 を取得・実行可能化（`aux_feeders/ieee34`。ただし**電圧調整器が
  欠落**しており末端が 0.92pu まで垂下＝忠実な IEEE34 ではない。標準 `examples/` には置かない）。
- IEEE13 の**3相7母線それぞれに1回ずつ**PV配置（`plans/hc13_hostingcapacity.yaml`, 復元抽出なし
  ＝真の独立標本）、pv_kw を 0→4000kW で掃引、各セルの母線最大電圧 max(V) から
  「過電圧ホスティング容量（max V ≤ 1.05 を保つ最大 pv_kw）」を算出。

## 実測結果（`results/` 未保存の解析値、再現はプランから可能）

| bus | Vmax0 | dVmax/dMW | 過電圧HC (kW) |
|---|---|---|---|
| 632/633/670/671/675/680/692 | 1.001 | 0.0025–0.0065 | **全て >4000（グリッド上限未達）** |

- **IEEE13 の母線最大電圧はほぼ 1.00**。強い変電所電源が電圧を保持し、4MW の PV でも過電圧
  （>1.05）を起こさない。予測ホスティング容量は 7.5–20 MW。
- したがって IEEE13 で観測された違反（前半の voltage_violation_rate 0.15–0.73）は**すべて低電圧
  （末端垂下 <0.95）由来**であり、候補1の前提「逆潮流で電圧上限超過」は**IEEE13 では再現しない**。
  PV は過電圧リスクではなく、むしろ低電圧の緩和として働く。
- IEEE34 は逆に規制器欠落で 0.92 まで垂下し、これも低電圧支配。

## 正直な結論：新規性は出なかった。構造的理由（実データに基づく）

1. **制御アルゴリズムの差込点が無い** → 新規手法を提案できない（新規性の主要源が塞がれている）。
   候補1の学問ギャップ（通信遅延ロバストな分散Volt-VAR制御）は制御器と時系列・動力学を要し、
   本framework（決定論・静的PF・PV注入のみ）では原理的に扱えない。
2. **到達できるのはホスティング容量の特性化のみ**で、標準試験フィーダーでのその定性挙動
   （強フィーダーは高HC、弱フィーダーは低電圧支配、PVは末端の低電圧を緩和）は**教科書的既知**。
3. クリーンな特性化ですら、母線ごとの kv/相数・電圧基準・過/低電圧の区別・規制器の有無など
   物理設定を正しく詰める手間が大きく、詰めた先の結果は既知の再現に留まる。
4. **集計メトリクス中心で母線別電圧が露出しない**ため、母線特徴量ベースの予測モデル等の
   踏み込んだ分析も標準経路では組みにくい（生JSONの max(V) 抽出という研究側解析が要る）。

## 新規性を可能にするには（framework 側の必要能力）

- 制御戦略の一級プラグイン点（Volt-VAR droop / consensus / MPC を差し替え比較できる interface）。
- 時系列・準動的シミュレーション（雲影の秒スケール確率過程＝候補1の核心）。
- 母線別電圧/枝潮流の結果露出（現状は集計 + フラット電圧ベクタのみ）。
- 忠実な複数フィーダー（規制器込みの IEEE34/123 等）を標準同梱。

これらは #28 とは別の、より根源的な follow-up（研究の**創発**を可能にする能力）であり、
「ツールを洗練しても、扱える問いの範囲を広げなければ新規性は出ない」という本試行の教訓。
