## 0. 結論

**「MVP で論文可能化の検証が通った」という自己判定（`review_record.md` §10.4
「投稿可 (Tier 2)」）は、独立に見直すと成立していない。**

正確な到達点は次の二段に分けられる:

- ✅ **検証できているもの**: gridflow の成果物生成パイプライン
  （sweep → comparison table → `gridflow export paper` で LaTeX 表 / CSV /
  matplotlib script / caption を再現可能に生成）は実際に動作し、
  report ↔ JSON の数値一致も現状取れている。
- ❌ **検証できていないもの**: MVP の本来の問い（policy §0
  「gridflow を使った研究者が査読論文を書けるか」）。中心定理の証明が破綻し、
  データ基盤の主張が誇大で、公平なベースラインに対する定量貢献が純負であり、
  しかも判定が自己採点の閉ループになっている。

したがって本 try が示したのは「**論文の体裁を持つ成果物を出力するパイプラインが
動く**」ことであって、「**査読に耐える科学的中身を出せる**」ことではない。

## 1. 検証プロセスが自己採点の閉ループ（構造的欠陥）

`report.md` を書いた仮想研究者が、`review_record.md` §7 で査読者 (M-1〜M-6) を
自演し、自分で反論し、§10.4 で「投稿可」と自己採点している。policy のフローは
Phase 2（仮想査読）→ Phase 3（プロダクトオーナー）だが、**独立な敵対的チェックが
一度も入っていない**。「論文を書けるか」を書いた本人が「書けた」と宣言している
だけであり、検証の体をなしていない。

## 2. 中心理論 Theorem 4 の証明が数学的に破綻

論文の目玉（Contribution #2「closed-form tail bound」）である Theorem 4 の
Proof sketch step 4（`theorems.md:64-70`）が内部矛盾:

- 分母を `Σ_{j=1}^N λ_j = N · N^{-1/α} / 2` としているが、全 DER のハザード総和は
  最悪 DER に支配されるため、このスケーリングは成り立たない。
- 仮にこの式を用いても `|S|N^{-1/α} / (N·N^{-1/α}/2) = 2|S|/N` であり、
  **主張の `N^{-(α-1)/α}` 因子はどこにも現れない**。続く行でこれに
  `N^{-1/α}·N^{1/α-1} = N^{-1}` を掛けて目的の形に整えているが、それなら
  結果は `2|S|/N²` になるはずで辻褄が合わない。
- すなわち **欲しい結論に向けて因子を後付けした証明**であり、論文の理論的背骨
  （heavy-tail closed-form bound）が立っていない。

## 3. Theorem 4′ は M11 を保証していない

revision の sharpened bound は「tier ranking = hazard ranking（完全識別 =
K→∞ 極限）」を仮定する（`theorems.md:120-122`）。これは**連続スコア法を
理想化した対象を bound しているのであって、有限 K の M11 ではない**。
Theorem 4 とは N 依存の関数形が別物（Th4: `|S|/N·N^{-(α-1)/α}` vs
Th4′: `((s+1)/N)^{(α+1)/α}`）であり、「3.5 倍 tight」（`theorems.md:118`）は
別物同士の比較。さらにハザードが Pareto 分位に従う (`λ_(i)=Λ(i/(N+1))^{1/α}`)
という根拠のないモデル仮定を追加で導入している。

## 4. 「実データ」の主張が誇大（M-4/M-5 自己応答の過大評価）

`run_heavy_sweep.py:68-115` より、ACN-Data 由来は **drop イベントの
タイムスタンプのみ**。手法が採点される最適化問題の構造 —
容量 `c_j` / コスト `g_j` / `τ` / exposure 集合 / burst 閾値 / active set —
は**すべて stationID からの `stable_hash` とシード乱数で合成**されている。

- `report.md` §7.3 (M-4) の「合成データ依存なし」「合成 simulator 不使用」は
  事実と異なる。drop 過程のみ実データ由来で、その上の SLA / coverage 問題は
  全合成。
- **P99 の結果が「合成属性割当の artifact」として丸ごと撤回された**（§6.5）こと
  自体が、結果が合成構造に支配されている証拠。撤回したのは P99 のみだが、
  主要指標 (commit_drop) も同じ脆弱性を共有する（寄与分解は未実施）。

## 5. 「再現性 ✅」認定自体が誤っていた（ゲート不全の証拠）

初版 Phase 2 自己レビュー（`review_record.md` §1 表 B）は再現性を ✅ とし、
§3 で「M11 が P99 で 5 手法中最低」「1.86× 改善」と報告していた。しかし当時の
実装はプロセスごとに salt される builtin `hash` を用いており**再現不能**で、
後に主要主張の一つ（P99 最低）が撤回された。

→ **自己レビューのゲートは、✅ を付けた当の項目（再現性）を実際には検出
できていなかった**。同じゲートが出す「投稿可」判定は信頼できない。

## 6. 公平なベースラインに対する定量貢献が純負

M11 が勝つのは M1/M10 のみで、これらは**構造上 履歴を使わない**手法
（`baselines_lit.py:142-225`）。**同じ情報量を持つ Fang/Singh に対しては、
主要指標で M11 が負け**（`report.md` §6.3: M11 17.53% vs Fang 14.76% /
Singh 14.09%、tail でも劣後）。正直な報告は誠実だが、定量的貢献は実質
「履歴を使えば使わないより良い」+「離散版は監査可能」に縮退する。後者の
理論的裏付け（Theorem 4）が §2 で崩れているため、残る差別化要素が宙に浮く。

## 7. policy §E-2（投稿先水準）との矛盾

policy §E-2 はトップ学会水準として **IEEE PES 標準フィーダー
（IEEE 13/34/37/123）を 2 つ以上**必須とし、自作・仮想のみを不可とする。
try16 は標準フィーダーを一切使わず、そもそも潮流解析ですらない合成 SLA 問題。
にもかかわらず自己レビュー（`review_record.md` §1 表 E）は E-2 相当を ✅。
基準の適用範囲に議論の余地はあるが、**ギャップを自己レビューが認識すら
していない**点が問題。

## 8. 軽微（品質シグナル）

- `run_heavy_sweep.py:10` のドキュメントは `alpha ∈ {0.5, 0.7}` だが、実際の
  sweep は `{0.10, 0.20}`（`run_heavy_sweep.py:260`, config と一致）。
  doc/code ドリフト。
- 「1,536 cells」は重複する複数 sweep（本比較 480 + K sweep 576 + 動的 480）の
  足し上げで、独立規模を過大に見せている。

## 10. 推奨（次アクション、新 try は不要）

1. **Theorem 4 の機械的検算**: step 4 の不等式連鎖を再導出するか、成り立たない
   なら主張を「empirical な N 依存改善の観測」に格下げする。
2. **実データ寄与 vs 合成寄与の分解実験**: 合成属性（cap/cost/τ/exposure）の
   シード集合を変えたときに主要指標 (commit_drop CI 分離) が保存されるかを示す。
   P99 と同じ artifact でないことを示せて初めて「実データ結果」と言える。
3. **独立 Phase 3 レビューの制度化**: 著者と別主体（または定理検算ツール）が
   ✅ を出すまで「投稿可」を確定しない。§5 の再発防止。
4. **policy §E との整合**: 本問題クラス（VPP standby、非潮流）に E-2 を
   どう適用するか policy 側で明文化するか、標準フィーダー要件を満たす問題に
   差し替える。
